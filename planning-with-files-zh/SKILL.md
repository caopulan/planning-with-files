# 文件化规划

像 Manus 一样工作：使用持久化 Markdown 文件作为你的“磁盘工作记忆”。

## 快速开始

在任何复杂任务开始前：

1. **在工作目录创建 `task_plan.md`**
2. **用复选框定义阶段**
3. **每个阶段完成后更新** - 勾选 [x] 并更新状态
4. **做决定前先阅读** - 刷新目标到注意窗口

## 三文件模式

对每个非平凡任务，创建三个文件：

| 文件 | 用途 | 何时更新 |
|------|---------|----------------|
| `task_plan.md` | 跟踪阶段与进度 | 每个阶段之后 |
| `notes.md` | 存放发现与研究 | 研究过程中 |
| `[deliverable].md` | 最终输出 | 完成时 |

## 核心流程

```
循环 1：创建 task_plan.md，写明目标与阶段
循环 2：研究 → 保存到 notes.md → 更新 task_plan.md
循环 3：阅读 notes.md → 产出交付物 → 更新 task_plan.md
循环 4：交付最终输出
```

### 循环详解

**每次重大行动前：**
```bash
读取 task_plan.md  # 将目标刷新到注意窗口
```

**每个阶段之后：**
```bash
编辑 task_plan.md  # 勾选 [x]，更新状态
```

**存储信息时：**
```bash
写入 notes.md     # 不要塞进上下文，存入文件
```

## task_plan.md 模板

任何复杂任务都先创建此文件：

```markdown
# 任务计划：[简短描述]

## 目标
[一句话描述最终状态]

## 阶段
- [ ] 阶段 1：计划与准备
- [ ] 阶段 2：研究/收集信息
- [ ] 阶段 3：执行/构建
- [ ] 阶段 4：审查与交付

## 关键问题
1. [需要回答的问题]
2. [需要回答的问题]

## 已做决策
- [决策]：[理由]

## 遇到的错误
- [错误]：[解决方案]

## 状态
**当前处于阶段 X** - [我现在在做什么]
```

## notes.md 模板

用于研究与发现：

```markdown
# 笔记：[主题]

## 来源

### 来源 1：[名称]
- URL：[链接]
- 要点：
  - [发现]
  - [发现]

## 综合结论

### [分类]
- [发现]
- [发现]
```

## 关键规则

### 1. 始终先创建计划
没有 `task_plan.md` 就不要开始复杂任务。这是硬性要求。

### 2. 先读再决定
做任何重大决策前，先读计划文件。让目标保持在注意窗口。

### 3. 行动后更新
每完成一个阶段，立刻更新计划文件：
- 用 [x] 标记已完成阶段
- 更新“状态”部分
- 记录遇到的任何错误

### 4. 存储，不要塞满上下文
大体量输出写入文件，不要塞进上下文。只把路径放在工作记忆里。

### 5. 记录所有错误
所有错误都写进“遇到的错误”部分。这会为以后任务积累知识。

## 何时使用此模式

**在以下场景使用三文件模式：**
- 多步骤任务（3 步以上）
- 研究任务
- 构建/创作内容
- 跨多个工具调用的任务
- 需要组织的任何事项

**以下场景可跳过：**
- 简单问题
- 单文件修改
- 快速查询

## 避免的反模式

| 不要 | 改为 |
|-------|------------|
| 用 TodoWrite 做持久化 | 创建 `task_plan.md` 文件 |
| 只说一次目标然后忘记 | 每次决策前重读计划 |
| 隐藏错误并反复重试 | 把错误记录到计划文件 |
| 把所有内容塞进上下文 | 把大内容存入文件 |
| 一上来就执行 | 先创建计划文件 |
